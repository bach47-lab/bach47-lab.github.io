Bachelor Thesis

A Comparative Analysis and Lifecycle-Oriented Tool Support for Threat Modeling in Cloud Environments with Privacy and Sovereignty Considerations

Supervisor: Prof. Dr. Ivan Gudymenko, Chair of Privacy and Security in Ubiquitous Computing

Cloud threat modeling is usually split into three disconnected exercises: security (STRIDE), privacy (LINDDUN), and digital sovereignty, which most existing tools ignore entirely. This thesis compares five threat modeling tools and approaches, builds a combined fourteen-category taxonomy that merges all three concerns into a single pass, and demonstrates it with a working, fully local Proof-of-Concept.

Comparative analysis

  • Evaluated Microsoft Threat Modeling Tool, OWASP Threat Dragon, IriusRisk, pytm, and LLM-based approaches across eight criteria spanning cloud support, privacy, sovereignty, and lifecycle coverage
  • Found that every evaluated tool scores zero on sovereignty awareness, and only one commercial tool covers the full threat lifecycle

Combined taxonomy

  • Built a sovereignty threat category from scratch (jurisdictional exposure, foreign access, vendor lock-in, data residency violation), grounded in GDPR Chapter V, the US CLOUD Act, the Schrems II ruling, and the Gaia-X Trust Framework
  • Combined it with STRIDE (security) and four LINDDUN categories (privacy) into one fourteen-sub-category taxonomy applicable in a single pass

Proof-of-Concept

  • Built a five-stage Python pipeline covering identification, categorization, prioritization, mitigation, and reporting, using Llama 3 via Ollama running fully locally with no external API calls
  • Kept risk scoring and mitigation mapping (NIST, CIS, GDPR, Gaia-X) deterministic and rule-based downstream of a single LLM call, so results stay reproducible

Key finding

  • On a sample AWS/Kubernetes architecture, the PoC identified 11 threats, including 3 sovereignty threats (e.g. jurisdictional exposure via a US-based email provider) that none of the five evaluated tools would have surfaced
  • Proposed a six-step continuous threat modeling workflow combining existing tools with the new sovereignty layer

2024 · Personal Project · github.com/bach47-lab/tls-cryptography-tool

TLS Handshake Simulator

A web app that runs a full TLS-like cryptographic handshake end to end in real time, built to make the abstract steps of PKI and key exchange concrete rather than just theoretical.

PKI & certificates

  • Generates RSA-2048 key pairs for both server and client
  • Signs certificates with SHA-256withRSA and verifies them, simulating a certificate chain

Key exchange & encryption

  • Performs Diffie-Hellman-2048 key exchange to derive a shared secret
  • Derives an AES-256-CBC session key from the shared secret for end-to-end message encryption and decryption

Delivery

  • Built as a Spring Boot backend with a vanilla JS frontend, so each handshake step is visible as it happens
  • Packaged as a Docker image published to GitHub Container Registry, runnable with a single command
